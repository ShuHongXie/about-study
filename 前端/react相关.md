### react fiber

1. react fiber中的fiber是一种数据结构
2. Fiber 把 “不可中断的长任务” 变成 “可中断的小任务”，并通过优先级调度保证交互优先，且只在最后一次性更新 DOM，所以不卡不闪。
   提交阶段是原子操作、不可中断的，所有 DOM 更新会一次性批量完成，浏览器只会在全部更新结束后才渲染页面，所以不会出现半拉子界面，也就不会闪烁。

### React Fiber 解决问题的核心机制（简洁版）

1. 因为 Fiber 用 “链表结构” 替代递归栈，支持渲染任务中断 / 恢复，所以 长任务被拆分为小任务分片执行，主线程每帧有空闲时间处理渲染 / 交互，不会卡顿；
2. 因为 Fiber 支持任务优先级调度（动画 / 交互 > 数据更新），所以 高优先级任务（如点击、动画）可抢占主线程，保证交互响应及时，避免卡顿；
3. 因为 Fiber 采用 “协调（可中断）+ 提交（不可中断）” 双阶段渲染，所以 仅在提交阶段一次性更新 DOM()，避免 DOM 碎片化更新导致的页面闪烁；
4. 因为 协调阶段仅标记 DOM 修改（无实际 DOM 操作），所以 即使渲染中断，界面也不会出现未完成的更新，消除闪烁。

### React Diff 算法核心对比过程（极简版）

React Diff 以「最小化 DOM 操作」为目标，基于同层对比、key 匹配、类型预判三大优化原则，将对比过程简化为三步核心逻辑：

1. 层级对比（树级）：只对比虚拟 DOM 树中同一层级的节点，不跨层级（比如根节点的子节点只和根节点的子节点比）。若节点类型不同（如 <div> vs <p>），直接销毁旧节点、重建新节点；若类型相同，进入下一步。
2. 节点类型对比（组件 / 元素级）：
   若是自定义组件：类型相同则复用组件实例、更新 props，再对比组件内部的虚拟 DOM；类型不同则销毁旧组件、重建新组件。
   若是原生 DOM 元素：只更新变化的属性（如 className、style），不重建元素，再对比其子节点。
3. 子节点对比（列表核心）：
   普通子节点：按顺序逐个对比，只更新内容 / 属性变化的节点；
   列表子节点：通过唯一 key 建立新旧节点映射，快速识别增 / 删 / 移动的节点（无 key 则按顺序暴力更新）。
